# Excel 订单素材下载工具——功能开发文档

> 产品名称：Excel 订单素材下载工具  
> 品牌标识：Power by 灵图（ipoddy.cn）  
> 目标平台：Windows 10 / Windows 11 x64  
> 目标用户：不具备 Python 或命令行使用经验的普通业务人员  
> 交付形式：单个可双击运行的 Windows EXE，用户无需安装 Python  
> 文档用途：供开发团队、Cursor 等代码助手直接读取并继续开发

---

## 1. 项目目标

读取用户选择的 `.xlsx` 订单文件，以表格中的每一条数据为一个订单，将文本信息、内嵌二维码以及多个下载地址中的素材整理到独立订单目录。

程序必须具备以下能力：

1. 自动跳过 Excel 第一行表头。
2. 每一条数据创建一个文件夹，文件夹名称使用该行第一列的值。
3. 将该行普通文本字段汇总到 `信息.txt`。
4. 提取该行“二维码”列中的 Excel 内嵌图片，保存为 `二维码.png` 或原始图片格式。
5. 处理所有列名中包含“下载地址”的字段，而不是只处理某一列。
6. 下载字段中的所有素材链接。
7. 下载完成后识别压缩包并自动解压。
8. 所有下载文件及解压结果直接平铺在订单目录中，不允许出现“通用”“效果图”等分类目录。
9. 单个链接失败或超时不能阻塞整个任务，必须记录失败并继续后续链接和订单。
10. 失败订单目录增加 `_下载失败` 后缀，并写入包含原始失败 URL 的 `link.txt`，方便人工下载。
11. 提供桌面 GUI，显示选择项、总体进度、当前订单、实时日志、失败原因和停止按钮。
12. 支持可见浏览器登录模式，解决需要 Google 登录态的下载。
13. 最终封装为 Windows x64 EXE，客户电脑不需要安装 Python、Playwright 或其他开发环境。

## 2. 输入文件约定

### 2.1 文件格式

- 输入格式：`.xlsx`
- 默认读取工作簿当前活动工作表。
- 第一行为表头，从第二行开始为订单数据。
- GUI 暂不要求用户选择工作表；命令行版本可以通过参数选择工作表。

### 2.2 典型字段

示例表头：

| 列名 | 用途 |
|---|---|
| `no` | 订单目录名称，必须位于第一列 |
| `二维码` | Excel 中嵌入的二维码图片 |
| `唯一一码` | 普通文本字段 |
| `sku` | 普通文本字段 |
| `品名` | 普通文本字段 |
| `数量` | 普通文本字段 |
| `通用下载地址` | 下载字段 |
| `效果图下载地址` | 下载字段 |

字段识别规则：

- 第一列始终作为订单号，不依赖表头是否为 `no`。
- 二维码列：表头必须精确等于 `二维码`。
- 下载列：表头只要包含 `下载地址` 四个字就必须处理。
- 其余非空字段写入 `信息.txt`。
- `二维码` 列和所有下载地址列不写入 `信息.txt`。

### 2.3 数据容错

- 第一列为空时，使用 `第N行` 作为目录名称。
- Windows 文件名非法字符 `< > : " / \ | ? *` 及控制字符替换为 `_`。
- 文件名末尾的句点和空格必须移除。
- 空白单元格不写入文本。
- 没有二维码时记录警告，但继续处理后续下载和订单。

## 3. 输出目录规范

### 3.1 正常订单

```text
用户选择的输出目录/
└── MEG2607150222/
    ├── 信息.txt
    ├── 二维码.png
    ├── design_1.png
    ├── design_2.png
    ├── design_3.png
    ├── design_4.png
    └── product_preview.png
```

### 3.2 下载失败订单

```text
用户选择的输出目录/
└── MEG2607150222_下载失败/
    ├── 信息.txt
    ├── 二维码.png
    ├── 已成功下载的文件
    └── link.txt
```

`link.txt` 要求：

- 使用 UTF-8 编码，推荐 UTF-8 with BOM，方便 Windows 记事本打开。
- 每行一个下载失败的原始 URL。
- URL 去重并保持首次出现顺序。
- 不记录已经成功下载的 URL。

### 3.3 平铺规则

所有下载内容直接进入订单根目录：

- 不创建 `通用` 目录。
- 不创建 `效果图` 目录。
- Google Drive 文件夹中的文件也必须平铺。
- 下载压缩包后，压缩包本身保留在订单根目录。
- 解压目录放在订单根目录，名称默认使用压缩包去除扩展名后的名称。
- 不保留 Google Drive 或远程文件夹的层级结构。

如果旧版本已经生成 `通用`、`效果图` 等分类目录，再次运行时应将这些目录的顶层内容迁移到订单根目录，并删除空分类目录。

### 3.4 同名冲突

严禁静默覆盖已有文件。

例如：

```text
image.png
image_2.png
image_3.png
```

目录重名也使用相同的递增后缀规则。

## 4. 文本与二维码处理

### 4.1 `信息.txt`

内容格式为每行一个字段：

```text
no: MEG2607150222
唯一一码: 150222-MEGCSSMT-2XL-3D
sku: MEGCSSMT-2XL-3D
品名: MEG正常领衬衫
数量: 1
```

要求：

- 保持 Excel 原始列顺序。
- 格式为 `表头: 值`。
- 空字段跳过。
- 使用 UTF-8 with BOM。
- 重跑时允许覆盖当前订单的 `信息.txt`，确保内容与 Excel 最新数据一致。

### 4.2 二维码

二维码通常不是单元格文本，而是锚定在单元格上的 Excel 内嵌图片。

实现要求：

1. 遍历工作表的图片对象。
2. 根据图片左上角锚点行列匹配“二维码”列及当前数据行。
3. 每行只保存一张二维码。
4. 默认文件名为 `二维码.<原始格式>`。
5. 如果无法确认原始格式，使用 PNG。
6. 没有图片时写日志并计入失败统计，但不能终止批处理。

注意：`openpyxl` 对图片使用部分内部属性，例如 `sheet._images`、`image._data()`，升级依赖时需要回归测试。

## 5. URL 提取规则

下载地址单元格可能包含：

- 一个普通 HTTP/HTTPS URL；
- 多个 URL；
- Google Drive 文件地址；
- Google Drive 文件夹地址；
- Printerval `folder-design` 页面；
- URL 查询参数中的多个 `design_urls`；
- 逗号分隔的资源路径；
- 相对资源路径。

基础要求：

1. 从单元格字符串中提取全部 HTTP/HTTPS URL。
2. 去重并保持出现顺序。
3. 单个 URL 的失败不能影响其他 URL。
4. 必须保留原始页面 URL，用于失败时写入 `link.txt`。

### 5.1 Printerval 特殊规则

对于 `printerval.com/folder-design` 类型地址：

1. 解析查询参数 `design_urls`。
2. `design_urls` 中可能包含用英文逗号分隔的多个路径。
3. URL 编码内容必须执行解码。
4. 相对路径补全为 `https://assets.printerval.com/...`。
5. 除下载按钮对应的设计图外，还必须解析页面 HTML，找到页面展示但没有下载按钮的产品效果图。

已确认的验收数据：

- 订单 `MEG2607150222` 应下载 5 张图片。
- 其中通常有 4 张来自 `design_urls`。
- 另外 1 张来自页面展示图，即使页面没有对应的 download 按钮也必须下载。

页面图筛选建议：

- 识别 `cdn.printerval.com/image/...` 的 PNG、JPG、JPEG、WEBP。
- 过滤 `/image/960x960/` 等缩略图路径。
- 优先保留无尺寸段的原图。
- 当前业务预期补充 1 张页面产品效果图；如页面结构变化，应通过测试样本调整选择规则。

## 6. 下载策略

### 6.1 处理顺序

每个 URL 按以下顺序尝试：

```text
直连或专用下载器
    ↓ 失败
根据浏览器回退设置决定是否启动 Playwright
    ↓ 仍失败
记录错误 + 写入失败 URL + 继续下一个链接
```

### 6.2 普通 HTTP/HTTPS

- 使用常见桌面浏览器 User-Agent。
- 单次请求设置明确超时，参考值 90 秒。
- 文件名优先从 `Content-Disposition` 获取。
- 其次使用最终响应 URL 的路径文件名。
- 仍无法得到名称时，按 Content-Type 推断扩展名。
- 必须跟随重定向。
- 使用流式写入，避免大文件完整载入内存。

### 6.3 Google Drive

支持：

- `https://drive.google.com/open?id=...`
- `https://drive.google.com/file/d/...`
- `https://drive.google.com/drive/folders/...`

推荐策略：

1. 优先使用 `gdown`。
2. `open?id=...` 先按单文件处理。
3. 单文件失败时，再将相同 ID 按文件夹地址尝试。
4. 下载到订单目录内的临时 staging 目录。
5. 下载完成后递归收集 staging 中所有文件，并平铺移动到订单根目录。
6. staging 目录必须自动清理。
7. `gdown` 失败后，根据浏览器模式决定是否使用 Playwright。

Google Drive 常见失败原因：

- 文件未设置为“知道链接的任何人可访问”；
- 下载频率限制；
- 必须登录 Google 账号；
- 文件确认页或病毒扫描页；
- `open?id=` 实际指向文件夹。

### 6.4 浏览器回退

提供三种模式：

| 模式 | 行为 |
|---|---|
| `visible` | 显示浏览器，允许用户登录；GUI 默认值 |
| `headless` | 无头浏览器自动尝试 |
| `off` | 不使用浏览器，直连失败即记录 |

具体要求：

- GUI 默认使用 `visible`。
- 可见模式优先尝试系统 Chrome，再尝试 Edge，最后尝试内置 Chromium。
- 无头模式优先使用 EXE 内置 Chromium。
- 浏览器上下文开启 `accept_downloads`。
- 浏览器下载保存到当前订单根目录。
- 浏览器回退单次最长等待建议为 45 秒。
- 找不到下载按钮时，尝试使用浏览器上下文中的登录态直接请求原 URL。
- 无论浏览器失败、超时或被用户关闭，都必须捕获异常并继续批处理。

Google 登录态：

- Windows 推荐保存到：
  `%LOCALAPPDATA%\LingTu\ExcelOrderDownloader\browser-profile`
- 不要将登录态保存在 EXE 临时解包目录。
- 登录态目录应跨运行复用。
- 不得将客户登录态打包进发布文件。

为避免任务卡死：

- Google Drive 在 `headless` 模式下，如果 `gdown` 已失败，可以直接跳过浏览器回退并记录 URL，因为无头环境无法人工完成登录。
- `visible` 模式允许打开浏览器等待用户登录，但仍必须有超时。
- 不允许无限等待下载事件。

## 7. 压缩包处理

支持格式：

- `.zip`
- `.tar`
- `.tar.gz`
- `.tgz`
- `.tar.bz2`
- `.tbz2`
- `.7z`

规则：

1. 下载完成后根据文件实际类型和扩展名判断是否为压缩包。
2. 原压缩包保留。
3. 解压到订单根目录下的同名目录。
4. 解压目录重名时增加 `_2`、`_3` 后缀。
5. 解压失败计入失败项，不能终止整个批处理。
6. 必须防止 Zip Slip / 路径穿越。
7. 解压前校验所有成员最终路径必须位于目标目录内。
8. TAR 解压使用安全过滤模式。

注意：7z 使用 `py7zr`，需要包含在运行依赖和 PyInstaller 隐式导入中。

## 8. 失败处理与重试

### 8.1 单项失败

以下错误都必须转换为可读日志，并继续后续任务：

- 网络超时；
- HTTP 错误；
- Google Drive 权限或限流；
- 页面结构变化；
- 浏览器启动失败；
- 浏览器等待下载超时；
- 文件写入失败；
- 解压失败；
- 二维码缺失。

日志至少包含：

- 当前订单号；
- 下载类别；
- 失败 URL；
- 失败原因；
- 是否启动浏览器回退；
- 是否已写入 `link.txt`。

### 8.2 订单失败标记

如果订单存在下载失败：

1. 先保留已经成功生成或下载的文件。
2. 写入 `link.txt`。
3. 将订单目录从 `订单号` 改名为 `订单号_下载失败`。

如果同名失败目录已经存在，不得覆盖，应使用安全的唯一名称策略。

### 8.3 再次运行

如果发现：

```text
订单号_下载失败/
```

且正常的：

```text
订单号/
```

不存在，则先将失败目录恢复为正常订单目录后重试。

全部失败链接本次均成功后：

- 删除旧的 `link.txt`；
- 保持目录名为正常订单号；
- 不再添加 `_下载失败`。

重跑不得覆盖已有下载素材；同名文件增加后缀。

## 9. GUI 功能要求

### 9.1 窗口

- 标题：`Excel 订单素材下载工具`
- 品牌文字：`Power by 灵图（ipoddy.cn）`
- 使用应用图标，但不在内容区重复展示大 Logo。
- 推荐初始尺寸：900 × 650。
- 推荐最小尺寸：760 × 560。

### 9.2 控件

必须包含：

1. Excel 文件路径输入框和“选择”按钮。
2. 输出目录输入框和“选择”按钮。
3. “仅处理前 N 条数据”复选框及数字输入框。
4. 浏览器回退模式下拉框：
   - `visible`
   - `headless`
   - `off`
5. “开始处理”按钮。
6. “停止”按钮。
7. “清空日志”按钮。
8. 当前状态文字。
9. `当前订单数 / 总订单数`。
10. 总体进度条。
11. 可滚动实时日志区域。

### 9.3 交互规则

- 未选择有效 `.xlsx` 文件时禁止启动并弹出错误提示。
- 测试条数必须为正整数。
- 开始后禁用“开始处理”，启用“停止”。
- 下载任务必须在后台线程运行，GUI 主线程不能卡死。
- 工作线程不能直接操作 Tkinter 控件；使用线程安全队列传递日志和状态。
- 点击停止后设置取消事件。
- 当前网络请求允许完成或超时，然后在安全检查点终止。
- 正常完成弹出成功提示。
- 存在失败项弹出警告，并提示查看日志及 `_下载失败` 目录。
- 停止任务不弹出错误。

### 9.4 进度

第一阶段至少实现订单级总体进度：

```text
[1/3] MEG2607150020
[2/3] MEG2607150222
[3/3] ...
```

建议后续升级为两级进度：

- 总体进度：已处理订单 / 总订单；
- 当前订单进度：已完成下载数 / 当前订单下载总数。

大文件下载建议额外展示：

- 已下载大小；
- 总大小；
- 当前速度；
- 剩余时间。

这些增强项不得改变“不阻塞 GUI”和“超时后继续”的基础要求。

## 10. 命令行接口

即使主要交付 GUI，也建议保留 CLI 便于测试和自动化。

建议接口：

```bash
python xlsx2files.py INPUT.xlsx \
  --output output \
  --sheet Sheet1 \
  --limit 3 \
  --browser-fallback visible \
  --browser-profile .xlsx2files-browser-profile
```

参数：

| 参数 | 说明 |
|---|---|
| `xlsx` | 必填，输入文件 |
| `-o, --output` | 输出目录，默认 `output` |
| `--sheet` | 工作表名称，默认活动表 |
| `--limit N` | 仅处理前 N 条数据 |
| `--skip-downloads` | 只生成目录、文本和二维码 |
| `--browser-fallback` | `off/headless/visible` |
| `--browser-profile` | 浏览器用户数据目录 |

退出码：

| 退出码 | 含义 |
|---|---|
| `0` | 全部成功 |
| `1` | 存在失败项或程序异常 |
| `130` | 用户请求停止或键盘中断 |

## 12. Windows 打包要求

### 12.1 构建目标

- 单文件 GUI 程序。
- 文件名：`Excel订单素材下载工具.exe`
- 不显示控制台窗口。
- 设置应用图标和 Windows 文件版本信息。
- 内置 Playwright Python 包及 Chromium headless shell。
- 内置项目 Logo。
- 客户电脑无需安装 Python。

### 12.2 推荐构建流程

```bat
py -3.12 -m venv .venv-win
call .venv-win\Scripts\activate.bat
python -m pip install --upgrade pip
python -m pip install -r requirements-build.txt

set PLAYWRIGHT_BROWSERS_PATH=%CD%\.playwright-browsers
python -m playwright install --only-shell chromium

python -m PyInstaller --noconfirm --clean xlsx2files_gui.spec
```

### 12.3 PyInstaller 配置重点

必须收集：

- `playwright` 的 datas、binaries 和 hidden imports；
- `.playwright-browsers`；
- `assets/lingtu-logo.ico`；
- `assets/lingtu-logo.png`；
- `gdown`；
- `py7zr`；
- `PIL._tkinter_finder`。

运行时资源路径必须兼容：

- 源码运行目录；
- PyInstaller 的 `sys._MEIPASS` 临时目录。

### 12.4 交付包

最终客户 ZIP 只包含：

```text
Excel订单素材下载工具.exe
客户使用说明.txt
SHA256.txt
```

客户不得收到：

- Python 源码；
- 虚拟环境；
- 构建脚本；
- 浏览器登录态；

## 13. 安全与隐私要求

1. 不上传 Excel 内容到自有服务器。
2. 所有订单数据默认只在客户本机处理。
3. 不记录 Google 账号密码。
4. 浏览器登录态仅保存在客户本机用户目录。
5. 日志不得输出 Cookie、Authorization Header 或浏览器存储内容。
6. 文件名必须过滤 Windows 非法字符。
7. 解压必须防止路径穿越。
8. 网络请求、浏览器等待必须设置超时。
9. 临时目录必须在完成或异常后清理。
10. 打包前清除开发者浏览器 profile，禁止将开发者登录态交付给客户。

## 14. 验收标准

### 14.1 基础解析

使用业务 Excel 前 3 条数据测试：

- 忽略表头；
- 创建 3 个订单目录；
- 目录名与每行第一列一致；
- 每个目录存在 `信息.txt`；
- 有二维码的行存在二维码图片；
- 文本不包含二维码和下载 URL 字段。

### 14.2 下载

- 两类下载地址都被处理。
- 一个单元格中多个 URL 全部被提取。
- 普通图片可下载。
- Google Drive 公开文件可下载。
- Google Drive 文件夹中的内容被平铺。
- Printerval `design_urls` 全部被下载。
- 无 download 按钮的页面效果图也被下载。
- `MEG2607150222` 最终应得到 5 张相关图片。
- 文件同名时不覆盖。

### 14.3 解压

- ZIP 下载后保留原文件并成功解压。
- 7z 成功解压。
- 解压结果位于订单根目录。
- 恶意路径压缩包被拒绝。
- 解压失败后继续处理下一个素材。

### 14.4 失败与重试

- 人为提供一个失效 URL。
- 程序在超时后继续下一链接和下一订单。
- 订单目录增加 `_下载失败`。
- `link.txt` 包含准确的失效 URL。
- 修复链接或网络后再次运行。
- 成功后删除 `link.txt` 并恢复正常目录名。

### 14.5 GUI

- 下载期间窗口可移动、可缩放，不显示“未响应”。
- 进度和日志持续更新。
- 停止按钮可以终止后续任务。
- `visible` 为默认浏览器模式。
- Google 登录页面可以显示并复用登录态。

### 14.6 Windows EXE

在一台没有安装 Python 的干净 Windows 10/11 x64 电脑测试：

- 双击 EXE 可以打开 GUI。
- 图标正确。
- 不弹出控制台窗口。
- 可以选择 Excel 和输出目录。
- 普通下载、浏览器回退和解压均可运行。
- 关闭并再次打开后，Google 浏览器登录态仍可复用。
